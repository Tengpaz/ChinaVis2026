# 京剧主题构成与组合模式可视分析

本项目完成 ChinaVis 2026 京剧剧本赛题中“从剧本中提取核心主题，分析不同剧本的主题构成及其组合方式”的可视化部分。网页采用纯静态实现，可直接打开 `visualization/index.html`，不需要安装依赖或启动构建工具。

## 如何查看

- 网页入口：`visualization/index.html`
- 数据说明：`2605_ChinaVis/README.md`
- 网页数据导出脚本：`visualization/scripts/export_vis_data.py`

如果 CSV 数据更新，可在项目根目录运行：

```bash
python3 visualization/scripts/export_vis_data.py
```

脚本会重新生成 `visualization/data/vis_data.js`，并保持网页可离线打开。

## 数据基础

- 剧目规模：1473 部京剧剧本。
- 数据来源：38 个剧本集合或名家剧本选集。
- 主题类别：9 类，分别为公案戏、神魔戏、爱情婚姻戏、战争戏、宫廷戏、家庭戏、武侠义气戏、历史戏、其他戏。
- 分类方法：采用现有百炼/Qwen 全量分类结果，每部剧最多保留 3 个主题及权重。
- 验证情况：`validation_llm_report.csv` 中 200 条抽样验证一致率为 0.84。

## 可视化设计

网页采用分析仪表盘结构，围绕赛题要求组织为 5 个视觉单元：

1. 总览指标  
   展示剧目数、集合数、主题数、最强主题、最强主题组合和平均活跃主题数。

2. 主题总览  
   条形图同时呈现主题总权重与主导剧目数量。历史戏总权重最高，为 682.73，并在 857 部剧中作为主导主题。

3. 主题共现网络  
   用节点表示主题，用边表示主题组合强度。边权来自每部剧主题权重的两两共现累加。

4. 集合主题热力图  
   行表示 38 个数据集合，列表示 9 类主题，颜色深浅表示集合内主题均值。点击单元格可联动筛选剧目。

5. 剧目探索与代表片段  
   支持按剧名、集合和主题筛选；每部剧用堆叠条展示主题构成。右侧展示当前主题关键词和代表片段，用于解释主题语义。

## 核心发现

- 历史戏是最强底色：主题总权重为 682.73，远高于其他主题，说明京剧剧本中历史人物、朝代叙事和历史事件具有基础性地位。
- 主题不是单标签结构：每部剧平均包含 2.64 个活跃主题，说明许多剧目同时承载历史、战争、家庭、爱情或公案等多重叙事功能。
- 最稳定的组合是“战争戏-历史戏”：共现权重 55.28，符合传统历史战争叙事中军事冲突推动情节的特点。
- “爱情婚姻戏-家庭戏”构成第二强组合：共现权重 31.48，显示婚恋主题常嵌入家庭伦理和亲属关系之中。
- “武侠义气戏-历史戏”“家庭戏-历史戏”“宫廷戏-历史戏”也有较强共现，说明历史叙事经常作为其他主题的背景框架。

## 答辩讲解思路

可以按“数据处理可信度 -> 总体分布 -> 组合结构 -> 来源差异 -> 文本证据”的顺序讲：

1. 先说明 1473 部剧本经过文本抽取、清洗、分段和 LLM 主题分类，抽样一致率 0.84。
2. 用总览条形图指出历史戏占据核心，但爱情婚姻、家庭、公案、武侠义气等主题也形成明显分层。
3. 用共现网络说明主题不是孤立出现，而是以若干稳定组合表达叙事功能。
4. 用集合热力图比较不同来源和名家剧本选集的主题偏向。
5. 最后点击主题展示关键词和代表片段，说明可视化结论有文本依据。
